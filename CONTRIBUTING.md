# Contributing

Thanks for contributing! Please follow these guidelines:

- Branches: create topic branches from `main` named `feat/...`, `fix/...`, or `chore/...`.
- Pull requests: open a PR for non-trivial changes and include a brief description of the change, testing steps, and any migration notes.
- Commits: write clear, small commits; prefer incremental changes.
- Tests: run the test suite locally and include or update tests for behavior changes.
- Review: ask for at least one reviewer for larger changes.

Agent policy
- The repository assistant (`.agent.md`) will create a branch and open a PR by default for non-trivial edits. It will not trigger CI, call external APIs, perform automated releases, or force-push without explicit consent.
